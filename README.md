## 🤖 External AI API Attribution

This project integrates third-party AI services to provide intelligent feedback features. All AI-generated content is clearly disclosed to users within the application.

| Provider | Service / API | Purpose in HappyASPA | SDK / Access Method | Official Docs |
| --- | --- | --- | --- | --- |
| ByteDance / Volcano Engine (字节跳动 / 火山引擎) | **Doubao (豆包) LLM** — `doubao-seed-2-0-lite-260215` | Analyse user speaking recordings and generate constructive English feedback | `volcenginesdkarkruntime` Python SDK | [Volcano Engine Docs](https://www.volcengine.com/docs/82379) |
| ByteDance / Volcano Engine (字节跳动 / 火山引擎) | **Volcano ASR** — BigModel Speech Recognition | Transcribe user audio recordings to text before LLM evaluation | REST API (`openspeech-direct.zijieapi.com`) | [Volcano Engine ASR Docs](https://www.volcengine.com/docs/6561) |
| ByteDance / Volcano Engine (字节跳动 / 火山引擎) | **TOS (Volcano Object Storage)** | Temporarily store user audio files in the cloud so that ASR and LLM services can access them via public URL | `tos` Python SDK (`tos-cn-beijing.volces.com`) | [TOS Docs](https://www.volcengine.com/docs/6349) |

> **Disclosure:** Feedback generated by the AI services above is clearly labelled as *AI-generated* within the Speaking module. Users are informed that scores and comments are produced by an external AI model and should be treated as reference material.

## 📂 Project Structure

The project follows a **Flask Blueprint** architecture to ensure modularity and scalability for the agile team.

```text
HappyASPA/
├── Dockerfile                  # 🐳 Docker image build instructions
├── docker-compose.yml          # 🐳 Container orchestration and volume mapping
├── app.py                      # Application Entry Point (App Factory, DB Init)
├── models.py                   # SQLAlchemy ORM Models
├── requirements.txt            # Project dependencies and packages
├── add_default_data.py         # CLI command script to add default data
├── blueprints/                 # Backend Logic (Modularized by feature)
│   ├── __init__.py
│   ├── auth.py                 # Authentication routes (/login, /register, /logout)
│   ├── dashboard.py            # Main Dashboard & Learning Progress Tracking
│   ├── forum.py                # Forum logic (List, View Post, Create Post)
│   ├── listening.py            # Listening practice logic
│   ├── orchard.py              # Orchard module logic 
│   ├── speaking.py             # Speaking practice logic (with audio/video upload)
│   ├── team.py                 # Team ranking logic
│   └── vocabulary.py           # Vocabulary module logic
├── migrations/                 # Database migrations (Flask-Migrate)
├── static/                     # Static assets (subtitles, videos, etc.)
├── templates/                  # Frontend Views (Jinja2 Templates)
└── uploads/                    # User uploaded files
```

## 🚀 Local Setup (For Developers)

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd HappyASPA
   ```

2. **Create a virtual environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Initialize the database**
   ```bash
   flask db upgrade
   flask add-default-data
   ```

5. **Run the local server**
   ```bash
   python app.py
   ```
   The application will be available at `http://127.0.0.1:5000/`.

## 🐳 Production Deployment (Docker)

For production or cloud servers (e.g., Aliyun ECS), the project uses Docker to ensure environment consistency.

1. **Ensure Docker and Docker Compose are installed on the server.**
2. **Clone the project and enter the directory.**
3. **Build and start the container in the background:**
   ```bash
   docker compose up -d --build
   ```
4. **Initialize production database (First time only):**
   ```bash
   docker exec -it happyaspa_web flask db upgrade
   docker exec -it happyaspa_web flask add-default-data
   ```
   *Note: User uploads and the SQLite database are safely mounted to local directories ( `./instance`) to prevent data loss upon container restart.*
